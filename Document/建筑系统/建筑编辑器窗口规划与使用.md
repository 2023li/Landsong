# 建筑编辑器窗口规划与使用

> 适用版本：2026-07-15 统一 `BuildingBase` 终态。创建入口：`Landsong/Building/建筑编辑器`；数值入口：`Landsong/Building/建筑数值导表工具`。

## 1. 窗口解决什么问题

一个正式建筑家族由 Family、ModuleSet、Presentation、唯一 Runtime Prefab 和 Catalog 登记组成。窗口把这五项收敛成一次原子创建，避免漏引用、重复 FamilyId、错误目录或重新引入等级 Prefab。

窗口不生成 C# 脚本，也没有“等待编译后续建”流程。所有普通 Runtime Prefab 都直接使用同一个具体 `BuildingBase`；创建完成后通过 ModuleSet 组合能力，通过 LevelConfiguration 配置各级数值。

## 2. 一次创建的产物

- `BuildingFamilyDefinition`：身份、占地、放置、施工、等级、条件和费用。
- `BuildingModuleSetDefinition`：玩法模块模板；允许为空。
- `BuildingPresentationDefinition`：施工/等级/样式到 View 的映射。
- 唯一 Runtime Prefab：`BuildingBase + BoxCollider2D + BuildingPresentationController + 空 ViewRoot`。
- 向标准 `BuildingCatalog.asset` 登记 Family。

任何目标资产已存在时，创建会拒绝覆盖。后续步骤失败时，窗口回滚本次创建的资产和 Catalog 登记。

## 3. 标准创建流程

1. 打开 `Landsong/Building/建筑编辑器`。
2. 在“创建建筑”填写身份与模块模板、放置规则、施工与初始等级、可选表现资源。
3. 点击“校验创建参数”，修正全部错误。
4. 点击“创建完整建筑资产组”。创建在当前编辑器帧内完成，无脚本编译阶段。
5. 切换到“编辑现有家族”，只维护 ModuleSet 结构、Presentation 和 Excel 不拥有的字段。
6. 在正式 Excel 增加该 Family 的公共行、连续等级和所有所需配置行。
7. 点击窗口右上角“导表”，执行全表校验与导入；未完成等级保持“开放=否”。
8. 执行窗口右上角“架构校验”。结构错误必须为零；缺 View 只会产生回填提醒。
9. 美术按《建筑Prefab与表现资源回填指南》回填。

## 4. 字段解释

### 身份与模块模板

| 字段 | 含义 |
| --- | --- |
| Family Id | 稳定主键，格式 `building.<snake_case>`；任务、科技、蓝图和存档使用它 |
| Display Name | 策划/UI 显示名 |
| Asset Name | 新建资产与 Runtime Prefab 的文件名前缀，不是脚本类名 |
| Module Template | 只决定初始 ModuleSet；不改变运行时类型 |
| 分类 / 图标 | 建造菜单分类与图标；分类只能使用人口、农业、工业、经济、科研、市政、军事、交通、装饰、奇观 |

模板当前有：

- 基础建筑（模块配置）：创建空 ModuleSet，之后按需要添加模块。
- 岗位生产建筑（模块配置）：预置 `BM_岗位运营`、`BM_资源产出` 以及相应 LV1 配置。
- 岗位维护生产建筑（模块配置）：按 `workforce -> maintenance -> production` 预置岗位、通用维护费、生产模块及相应等级配置。

### 放置与地图规则

- Footprint 是整个家族全生命周期唯一占地，升级不可改变。
- Placement Costs 在放置确认时一次扣除，与逐回合施工费用不同。
- Required Terrain Keys 要求 footprint 的每一个格子都同时具备全部 Key；例如采石场填写 `陆地` 与 `石矿`，因此 3×3 的 9 格必须全在矿脉上。
- “占地内至少一格需要的 Key”只用于确实允许局部覆盖的其他机制，不能用于采石场。
- Movement Resistance、Blueprint、Sort Order、Max Build Count 均属于家族静态配置。
- Resource Provider Point 表示该家族提供连接能力；具体运行门槛仍由模块能力决定。

### 施工与等级

- Construction Turns 只保存非负的“本回合费用”和“本回合产出”，写入 Family；施工表现由独立的“施工视图模式”与 View 字段写入 Presentation。施工是 Stage，不是 LV0。
- “单一施工视图”在整个施工阶段保持同一个 View；“逐回合施工视图”会生成与 Construction Turns 一一对应的 View 列表，完成当前回合后切换到下一项。两种模式互斥。
- Initial Level Count 是首次创建时生成多少个连续运营等级定义，不是“建筑初始等级”。最小为 1，生成的第一个等级始终是 LV1。
- 新生成的 LV1 可用；之后新增的未完成等级应明确关闭。

### 表现资源

- 单一模式只显示一个 Construction View Prefab；逐回合模式只显示逐回合 View 列表。逐回合缺项时使用统一占位表现，不会读取单一模式字段。Placement Preview 与 Default Operational View 同样是独立纯 View Prefab，可暂时为空；放置预览留空时回退到当前样式的 LV1 运营 View。
- Styles 用于树种/皮肤等不改变玩法的玩家可选变体。
- 不允许把施工、LV1～LVN 全部美术节点塞进 Runtime Prefab。

### 岗位生产模板

这里配置岗位上限、初始工人、吸引力、招工/补贴、生产周期、产出物和工人档位。维护模板还配置维护物品与每回合维护费。创建后这些值进入 ModuleSet 与 LV1 LevelConfiguration；运行时状态由模块克隆持有。

金币、食物、产出和奖励等物品都通过 `ItemDefinition` 资产选择器配置，窗口不接受手填 ItemId。运行时由模块读取资产的稳定 ItemId 调用库存，存档仍保存 ItemId。

## 5. 正式数值配置

正式源表位于项目根目录的 `ConfigSource/Buildings/建筑数值策划表.xlsx`，不会进入 Unity AssetDatabase。建筑编辑器右上角：

- “数值文档”打开权属与导表说明；
- “数值表”打开 Excel；
- “导表”打开全表校验、实际差异预览和增量执行窗口；读取与校验始终覆盖全表，执行时只保存变化资产。

Excel 管理公共数值、消耗、施工、等级、LevelConfiguration 和已接入的模块默认值。导表字段对应的 SO 只作为运行时生成结果，不再作为策划手工源。导表器不自动创建未知 Family 或 ModuleSet 结构，任一错误会阻止全部写入。

## 6. 创建后的功能配置

创建新建筑后通常不编写代码：

1. 在 ModuleSet 末尾添加已有模块。
2. 若模块有等级参数，在每个开放等级添加对应 LevelConfiguration。
3. 确保被依赖模块排在使用者之前，例如 `workforce` 位于 `production`、`crop`、市场结算和 `operational_experience` 之前；有维护费时 `maintenance` 位于经验或生产之前。
4. 在 Presentation 选择单一或逐回合施工视图模式并回填对应字段，再配置可选放置预览视图、默认运营视图和必要的等级/样式 ViewMapping。
5. 执行架构校验并进入 Play Mode 验证。

只有现有模块无法表达全新机制时才编写代码。新代码应是带稳定 `[BuildingModuleId]` 的 `BuildingModuleBase` 子类，并按需要实现生命周期、自动回合、存档、UI 和能力接口；仍然不创建建筑类。

## 7. 模块编辑器

ModuleSet 使用托管引用列表。列表项标题显示“中文模块名 `[稳定 ModuleId]`”。点击 `+` 后必须从类型菜单选择具体模块；“未选择模块”只是新空槽位，选择类型后才会显示配置字段。

同一 ModuleId 在一个家族中只能出现一次。模块是否启用会影响能力查询、生命周期和等级配置依赖；不要通过禁用一个被 LevelConfiguration 引用的模块来临时绕过玩法。

## 8. Runtime Prefab 固定结构

```text
<AssetName>Runtime
├─ BuildingBase
├─ BoxCollider2D
├─ BuildingPresentationController
└─ ViewRoot
```

`ViewRoot` 在资产中必须为空。Runtime Prefab 不允许 Renderer、Animator、家族派生脚本、等级数值或模块模板。美术对象只从 Presentation 在运行时装载。

## 9. 校验与维护

`Landsong/Building/Validate Final Architecture` 检查：

- FamilyId、Catalog 与 Runtime Prefab 一对一；
- 根组件直接是统一 `BuildingBase`，项目中不存在普通具体派生类；
- Family、ModuleSet、Presentation 和反向引用完整；
- LV1 可用、等级连续、配置 ID 唯一；
- LevelConfiguration 所需模块存在且启用；
- ModuleId 唯一、依赖存在且顺序正确；
- 条件化范围效果的生效等级存在，最低工人不超过该级岗位上限；
- 模块与 LevelConfiguration 的物品引用有效，所引用 `ItemDefinition` 具有稳定 ItemId；
- Runtime Prefab 的空 ViewRoot 与无内嵌表现；
- View Prefab 不含玩法、碰撞或建筑运行时组件；
- 旧等级 Prefab/脚本不存在，所有正式资产均被 Catalog 引用。

`Landsong/Building/Normalize Runtime Prefabs` 用于迁移或大规模字段调整后重新保存全部统一 Runtime Prefab，清除旧序列化字段。普通内容编辑不需要反复执行。

## 10. 常见问题

- 新模块槽位无法配置：点击槽位标题或类型选择菜单，先选具体模块类型。
- 模块名称只显示“元素 0”：确认使用项目的托管引用 Drawer；正常标题应是模块中文名和 ModuleId。
- 创建按钮不可用：等待 Unity 当前脚本编译结束；创建本身不会触发脚本生成。
- 建筑显示占位图：结构仍然有效，按 Presentation 回填独立 View。
- 升级时报缺模块：目标级 LevelConfiguration 与 ModuleSet 不一致，补模块或删除错误配置。
- 想增加家族独有行为：新增单家族模块，不新增 `BuildingBase` 派生类。
- 导表按钮不可用：确认正式源为项目根目录 `ConfigSource/Buildings/建筑数值策划表.xlsx`，点击“重新读取并校验”，修复全部错误。
- Excel 已改但 Unity 未变化：保存并关闭 Excel 的占用，再重新读取；只有“执行导入”才会写入 SO。
- 新模块没有对应表：先确定它是等级配置还是模块默认值，再增加专表、解析模型、依赖校验和应用映射。

## 11. 提交标准

- 创建/修改后架构校验零错误。
- 新等级、费用、模块数值写入正式 Excel，并通过全表导入。
- 新表现同步美术回填清单。
- Play Mode 至少验证建造、施工完成、升级、保存/读取和该模块的主要失败路径。
