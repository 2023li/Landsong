# 建筑正式数值表与导表工具

> 生效日期：2026-07-15
> 正式策划源：[建筑数值策划表.xlsx](../../ConfigSource/Buildings/建筑数值策划表.xlsx)
> Unity 入口：`Landsong/Building/建筑数值导表工具`

## 1. 最终结论

建筑数值 Excel 已经是项目的正式策划源，不再是手工维护的资产快照。

- 策划在 Excel 修改导入范围内的数值。
- 导表器将 Excel 单向写入现有 `BuildingFamilyDefinition`、`BuildingLevelDefinition`、LevelConfiguration、模块默认值和 Runtime Prefab 的三个纯数据字段。
- 游戏运行时仍读取 Unity 资产，不在玩家构建中读取或解析 Excel。
- 导入字段对应的 SO 值是生成结果，不允许再与 Excel 双向手工维护。
- Prefab 结构、ModuleSet 模块类型、Presentation 的视觉样式和美术引用继续在 Unity 建筑编辑器中维护；运营 ViewMapping 的等级轴由 Excel 自动同步。

因此，Excel 固定放在项目根目录的 `ConfigSource`：它仍随仓库版本控制，但不会进入 Unity AssetDatabase。导表器使用项目相对路径直接读取磁盘文件，游戏运行时不会读取或解析该 `.xlsx`。

## 2. 正式位置与入口

```text
ConfigSource/Buildings/建筑数值策划表.xlsx
```

可从三个位置进入：

1. Unity 菜单 `Landsong/Building/建筑数值导表工具`。
2. `Landsong/Building/建筑编辑器` 右上角“数值表”。
3. `Landsong/Building/建筑编辑器` 右上角“导表”。

建筑编辑器右上角“数值文档”打开本文，用于查看权属和流程；“数值表”打开 Excel 本体。

### 2.1 建筑分类

正式分类固定为以下十类，代码枚举、建造栏顺序和 Excel 下拉选项必须保持一致：

| 分类 | 用途 |
| --- | --- |
| 人口 | 住宅、人口容量与人口服务 |
| 农业 | 种植、养殖、捕鱼与食物获取 |
| 工业 | 资源采集、加工与制造 |
| 经济 | 市场、贸易与财政 |
| 科研 | 研究与科技产出 |
| 市政 | 治理、公共服务与城市核心 |
| 军事 | 防御、驻军与训练 |
| 交通 | 道路、运输与通行设施；道路专用放置逻辑以该分类判断 |
| 装饰 | 景观与非核心装饰 |
| 奇观 | 唯一性或里程碑大型建筑 |

当前家族映射：王宫/警局→市政、居民房→人口、农田/捕鱼小屋→农业、伐木小屋/采石场→工业、市场→经济、仓库→人口|经济|工业、医院→人口|市政、泥路→交通、树木/雕塑→装饰。当前没有正式归入科研、军事和奇观的家族。

当前工作簿 `SchemaVersion` 为 `6`：版本 6 在版本 5 基础上增加通用运营经验等级配置，并让范围效果支持精确运营等级、最低工人以及 `medical` / `security` 格子类型。旧版本源表不能继续导入，也不提供兼容分支。

## 3. 工作表职责

| 工作表 | 导入 | 组合键/职责 |
| --- | --- | --- |
| `说明` | 元数据 | `B4` 固定保存 `SchemaVersion`；不要移动 |
| `建筑家族` | 是 | 一行一个现有 `FamilyId`；公共数值、逐格/整片地形要求、菜单、蓝图自动条件、资源点和行动力 |
| `放置消耗` | 是 | `FamilyId + ItemId` |
| `施工消耗` | 是 | `FamilyId + 回合序号 + ItemId` |
| `施工产出` | 是 | `FamilyId + 回合序号 + ItemId`；与同回合消耗原子结算 |
| `运营等级` | 是 | `FamilyId + Level`；等级是否开放及升级条件 |
| `升级消耗` | 是 | `FamilyId + 目标Level + ItemId` |
| `等级_固定人口` | 是 | `PlayerHomeLevelConfiguration` |
| `等级_库存容量` | 是 | `BuildingInventoryLevelConfiguration` |
| `等级_科技点` | 是 | `BuildingTechnologyPointLevelConfiguration` |
| `等级_住宅` | 是 | `ResidentialHousingLevelConfiguration` |
| `等级_岗位` | 是 | `BuildingWorkforceLevelConfiguration` |
| `等级_生产` | 是 | `BuildingProductionLevelConfiguration`；一行一个产出档位 |
| `等级_维护费` | 是 | `BuildingMaintenanceLevelConfiguration`；对应通用 `BM_维护费` |
| `等级_运营经验` | 是 | `BuildingOperationalExperienceLevelConfiguration`；工人门槛、每回合经验和下级累计经验要求 |
| `等级_捕鱼稀有` | 是 | `FishingHutLevelConfiguration` |
| `等级_仓库` | 是 | `WarehouseLevelConfiguration`；容量、维护费、经验、满员奖励与维护失败惩罚 |
| `模块_市场` | 是 | `BM_市场资源结算` 的模块级默认值 |
| `模块_树木` | 是 | `BM_树木采集` 的模块级默认值 |
| `模块_范围效果` | 是 | `FamilyId + EffectId`；效果类型、目标、生效 Level、最低工人、范围、数值与叠加规则 |
| `模块_作物` | 是 | 一行一个 `FamilyId + CropId`；当前正式数据为空 |
| `作物_种植消耗` | 是 | `FamilyId + CropId + ItemId` |
| `作物_收获产出` | 是 | `FamilyId + CropId + ItemId` |
| `作物_自动收获` | 是 | 作物模块级自动收获费用 |
| `科技_全局生产Buff` | 是 | 科技条件激活的建筑生产固定加成；图标与效果结构由 Unity Buff 资产维护 |
| `视觉样式` | 否 | 美术回填清单；不会覆盖 `BuildingPresentationDefinition` |
| `数据字典` | 否 | 稳定 ID、显示说明和权属索引 |
| `汇总检查` | 否 | Excel 内快速体检；不替代 Unity 校验 |

这种拆表方式与建筑架构一致：公共数据放公共表，等级能力按 LevelConfiguration 类型拆表，非等级模块规则放模块表。新增机制时增加对应配置表和导入映射，不向“建筑家族”主表堆大量只对少数建筑有效的空列。

施工回合数量、消耗和产出属于正式 Excel 数值；单一/逐回合施工视图模式及其施工 View 映射属于 Unity Presentation 表现配置，不写入 Excel。运营 ViewMapping 的 View 引用同样属于 Unity，但槽位结构由 `Excel 运营等级 × Unity 视觉样式` 自动生成。导表只在等级变化导致矩阵结构变化时把 Presentation 列为影响资产，并只补充、排序或移除空槽位；已有匹配槽位的美术引用不会被覆盖。待移除槽位仍配置 View 时整次导入会被阻止。

## 4. 策划标准流程

1. 关闭或保存 Excel 中正在编辑的工作簿。
2. 只修改黄色输入单元格；蓝色列为公式或检查结果，不参与导入。
3. 不修改工作表名、第 4 行列名、`说明!B4` 的位置或 `FamilyId`。
4. 回到 Unity，打开 `Landsong/Building/建筑数值导表工具`。
5. 点击“重新读取并校验”。
6. 修复全部错误。警告不会自动阻止导入，但必须阅读。
7. 查看“实际差异预览”，确认真正发生变化的 Family、ModuleSet、Presentation、Runtime Prefab 或独立数值 SO。Presentation 只应在运营等级变化需要同步固定 ViewMapping 矩阵时出现。
8. 有差异时点击“执行导入”。一次增量导入作为一个 Undo 操作记录；无差异时按钮保持禁用。
9. 执行 `Landsong/Building/Validate Final Architecture`，并在 Play Mode 验证受影响玩法。

## 5. 导入的原子性

导表分为两个阶段：

```text
读取 XLSX -> 建立内存模型 -> 全表校验 -> 全表差异比较
                                      |
                                      +-- 有错误：0 个资产写入
                                      +-- 无差异：0 个资产写入
                                      +-- 有差异：一次事务中仅写入变化资产
```

差异比较覆盖 Family 数值、由运营等级驱动的 Presentation 固定 ViewMapping 矩阵、ModuleSet 中由 Excel 管理的默认值、Runtime Prefab 的三个数值字段、独立空间效果 SO 与全局 Buff SO。未变化资产不会注册 Undo、不会标记 Dirty，也不会保存。执行期若发生异常或终态架构校验失败，工具会回滚本次所有修改并重新保存回滚后的资产。不会留下“部分家族已更新、部分未更新”的中间状态。

## 6. 当前校验规则

导表器至少检查：

- `SchemaVersion` 与导表器版本一致；
- 正式表包含项目中全部现有建筑家族；
- 未知或重复 `FamilyId` 被拒绝，导表器不自动创建结构资产；
- 等级从 LV1 连续，LV1 必须开放；
- 成本、作物、产出和等级组合键不重复；
- `ItemId` 唯一解析为真实 `ItemDefinition`；
- 科技条件唯一解析为 `TechnologyDefinition`；
- 施工回合不超过家族施工回合数，升级费用不允许写在 LV1；
- 施工消耗与产出都必须为正数量；数据分栏但运行时通过同一个库存事务结算；
- 每个 LevelConfiguration 所需的模块存在且启用；
- 拥有等级型模块的家族，每个运营等级都有完整配置；
- 生产必须有对应岗位，工人阈值不得超过岗位上限；
- 通用维护费必须匹配 `maintenance` 模块和每级配置，且模块顺序必须位于生产前；
- 通用运营经验必须匹配 `operational_experience` 模块和每级配置；经验工人门槛不得超过本级岗位上限，模块必须位于岗位和维护费之后；
- 捕鱼稀有产出的工人数、概率和数量合法；
- 仓库的容量、经验和奖励工人阈值不得超过对应等级的岗位上限，维护物品必须可解析，所有数值不得为负；
- 市场、树木、范围效果、作物配置只能写给拥有相应模块的家族；
- 每条范围效果必须匹配 Unity 中已存在且唯一的 `EffectId`，类型、目标、生效 Level、最低工人、叠加规则、范围和数值必须合法；
- 美化、医疗和治安效果只能以 `cell` 为目标；生效 Level 必须存在且开放，最低工人不得超过该级岗位上限；关闭“影响自身占地”时半径必须至少为 1；
- 人口、工人、回合、概率、奖励和成本满足各自范围。
- 蓝图自动条件与全局 Buff 条件中的科技必须唯一解析；全局生产 Buff 必须匹配已有 BuffId、目标家族和物品。

通过 Excel 内公式不等于通过 Unity 导表器。Unity 校验负责跨资产引用和架构依赖，是最终导入门槛。

## 7. 数据权属边界

### Excel 拥有

- 家族公共玩法数值；
- 放置、施工消耗与施工产出、升级消耗；
- 地形 Key、蓝图自动解锁条件；
- 运营等级开放状态与受支持的升级条件；
- 当前已接入的所有 LevelConfiguration 数值；
- 仓库等级配置、通用维护费、通用运营经验，以及市场、树木、范围效果、全局生产 Buff 和作物模块的策划默认值；
- Runtime Prefab 上的资源提供点、提供优先级和行动力预算三个标量。

### Unity 拥有

- `FamilyId` 的创建和重命名决策；
- Runtime Prefab 的组件、层级和引用结构；
- ModuleSet 中模块的类型、顺序和启用状态；
- Presentation、Style、Icon、View Prefab 与动画资源；
- 建造菜单显示条件和 Excel 尚未支持的复杂 `GameCondition`；科技型蓝图条件由 Excel 管理；
- 全局 Buff 的图标和效果类型结构；Excel 只拥有已接入效果的数值与目标引用；
- 作物图标。导入作物时会按 `CropId` 保留已有图标。

## 8. ID 规则

Excel 必须使用稳定键，不使用资源文件名或显示文案做关联：

- 物品：填写 `ItemDefinition.ItemId`，例如 `金币`、`原木`、`鱼`。
- 科技升级条件：`technology.<TechnologyId>`，例如 `technology.TN_3_1_启蒙`。
- 蓝图解锁条件与全局 Buff 条件使用同样的 `technology.<TechnologyId>` 格式。
- 无升级条件：`none`。
- 保留某个现有等级的复杂条件：`keep`。新等级不允许使用 `keep`。
- 作物：`FamilyId + CropId`。
- 范围效果：`FamilyId + EffectId`；树木使用 `building.tree + beauty.tree`，雕塑使用 `building.sculpture + beauty.sculpture`，医院使用六条 `medical.hospital.*`，警局使用六条 `security.police_station.*` 稳定 ID。
- 范围效果稳定值：`Kind` 使用 `beauty` / `medical` / `security` / `production_percent`，`TargetFilter` 使用 `cell` / `any_building` / `farmland`，`StackingRule` 使用 `additive` / `no_stack` / `highest_value`。`生效Level=0` 表示全部等级，正数表示精确等级；`最低工人=0` 表示无岗位门槛。
- 视觉：`FamilyId + StyleId`，但视觉表当前不参与数值导入。

## 9. 新建筑与新模块

导表工具只更新现有结构，不创建未知建筑或新模块。标准顺序是：

1. 用建筑编辑器创建 Family、ModuleSet、Presentation 和唯一 Runtime Prefab。
2. 在 ModuleSet 添加并排序需要的模块；范围效果还必须创建 `BuildingSpatialEffectDefinition` 资产并由 `BM_空间效果源` 引用。
3. 为新的 LevelConfiguration 类型增加运行时代码、校验映射、Excel 专表与导入映射。
4. 在 Excel 增加该家族的公共行、连续等级行和全部所需配置行。
5. 通过导表、架构校验和 Play Mode 验证。

## 10. 当前内容缺口

以下是明确的内容生产待办，不是导表架构缺陷：

1. 王宫 LV2～LV4 正式数值、科技条件和升级费用。
2. 居民房 LV2～LV4 正式成长曲线、条件和费用。
3. 捕鱼小屋 LV2 与黄金鱼启用时机；当前建筑仍标记为开发未完成。
4. 农田作物、种植费用、收获产出和自动收获费用；当前正式作物表为空。
5. 各家族正式菜单排序；当前仓库为 30，其他家族仍需统一规划。
6. 树木普通拆除是否应该发放采集奖励的最终平衡决策。
7. 视觉 Style、医院和警局等家族的 Icon 与 View 回填。

仓库是 `building.warehouse`，正式数值全部位于公共费用表、`运营等级`、`等级_岗位` 和 `等级_仓库`。LV1/LV2 的经验阈值分别为累计 30/50；LV3 达到 5 工人时从 3 格提升为 4 格。维护费使用金币，失败时本回合不加经验并触发 -10 岗位吸引力，成功支付后解除。

树木美化已是正式数值：`beauty.tree`、曼哈顿半径 1、每格 +1、排除自身占地、`highest_value` 取最高。对 1×1 树木，这恰好是上、下、左、右四格；施工阶段不生效，完成进入运营后生效。

雕塑是 `building.sculpture`，放置消耗 100 金币，施工 1 回合并消耗 20 石头，只有 LV1。`beauty.sculpture` 同样使用半径 1、每格 +1、排除自身占地和 `highest_value`；正式表的两个 Style 仅作美术回填跟踪，不参与玩法数值导入。蓝图条件由 `建筑家族.蓝图解锁ConditionId` 指向 `TN_4_3_石工术`。

采石场是 `building.quarry`。`建筑家族.地形Keys` 固定为 `陆地|石矿`，表示 3×3 的每一个占地格都必须同时包含两个 Key；`占地内至少一格Keys` 必须留空。蓝图由 `TN_5_3_采矿` 解锁；施工三回合的消耗和产出分别写入 `施工消耗`、`施工产出`。LV1～LV3 的岗位、维护费和工人数产量分别写入 `等级_岗位`、`等级_维护费`、`等级_生产`。`buff.quarry.masonry` 写入 `科技_全局生产Buff`，由石工术激活并在基础石头产量大于 0 时每次固定 +1。

医院是 `building.hospital`，分类为 `人口|市政`，2×2、施工 2 回合，放置消耗 20 金币，施工依次消耗 10 原木和 10 石头。LV1～LV3 的岗位上限为 4/8/16，维护费为 1/3/6 金币；通用运营经验写入 `等级_运营经验`，经验累计跨等级保留，LV1/LV2 的下级门槛为 30/100。LV2 与 LV3 分别由 `TN_2_6_医学`、`TN_2_13_生物学` 解锁，升级费用为 300/1000 金币。六档医疗规则写在 `模块_范围效果`：按精确运营等级和最低工人数选择半径 5/8/12/18 与医疗值 1/2/3，排除自身占地并使用 `highest_value`，因此同格只取最高医疗值。

警局是 `building.police_station`，分类为 `市政`，2×2、施工 2 回合，放置消耗 20 金币，施工依次消耗 10 原木和 10 石头；蓝图由 `TN_1_5_法律` 解锁。LV1～LV3 的岗位上限为 4/8/16，维护费为 1/3/6 金币；通用运营经验写入 `等级_运营经验`，经验累计跨等级保留，LV1/LV2 的下级门槛为 30/100。LV2 与 LV3 分别由 `TN_2_8_教育制度`、`TN_4_15_信息化` 解锁，升级费用为 300/1000 金币。六档治安规则写在 `模块_范围效果`：按精确运营等级和最低工人数选择半径 5/8/12/18 与治安值 1/2/3，排除自身占地并使用 `highest_value`，因此同格只取最高治安值。

占地全等级固定。任何“升级改变占地”的需求不得直接写入 Excel，必须新建家族或重新评审领域规则。
