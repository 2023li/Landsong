# 建筑说明

> 当前事实版本：2026-07-13。建筑领域重构已经落地；旧等级 Prefab、施工 Prefab 和等级脚本已删除。

## 1. 当前架构

所有建筑都使用“一个家族定义 + 一个 ModuleSet + 一个 Presentation + 一个轻量 Runtime Prefab”。Runtime Prefab 直接使用统一且密封的 `BuildingBase`，没有家族脚本；玩法由 ModuleSet 组合。施工和升级在同一实例上推进：

```text
Construction -> Operational/LV1 -> LV2 -> ... -> LVN
```

当前 `BuildingCatalog` 注册 8 个家族，Runtime Prefab 位于 `Assets/Landsong/Objects/Prefabs/BuildingsRuntime`。原 `Assets/Landsong/Objects/Prefabs/建筑` 中不符合规范的等级 Prefab、施工 Prefab、仓库壳和云中城壳均已移除。

## 3. 各家族行为

### 王宫

- 运营 LV1 提供 10 人口、5 个库存格、每回合 1 科技点。
- 是低优先级 Resource 提供点，市场可用时消费者优先市场。
- 数量上限 1；地图中的固定模板直接以运营态注册，不重新走玩家施工流程。
- LV2～LV4 复制了 LV1 的完整能力配置以防止能力断层，但 `configured=false`，没有数值和费用设计前不能升级。

### 居民房

- 放置立即支付 30 金币、1 石头、1 原木。
- 施工三回合依次支付：10 泥土、10 石头、10 原木。
- 完工后同一实例进入 LV1，初始人口 2、上限 5。
- 每回合按人口消耗等量蔬菜；成功消耗累计 3 回合后人口 +1。
- 连续 3 次消耗失败后人口 -1；人口归零后荒废。
- 满人口时每 5 个成功运营回合提供等于当前人口的金币税收。
- 必须能连接 Resource 提供点；高等级配置沿用完整住宅逻辑，不会再丢失低等级能力。
- LV2～LV4 当前关闭，费用和成长曲线待策划。

### 伐木小屋

- LV1：岗位 3，初始 2 工人；每 3 回合结算，2 工人产 1 原木、3 工人产 2 原木。
- LV2：岗位 5；保留原产量档并增加 5 工人产 4 原木。
- 升入 LV2 需要科技 `TN_3_1_启蒙`，支付 30 金币。
- 岗位和生产进度都由稳定模块保存，升级后不丢工人、不重建对象。

### 捕鱼小屋

- LV1：岗位 3，初始 0 工人；每 3 回合结算，2 工人产 1 鱼、3 工人产 2 鱼。
- 特殊黄金鱼参数已保留为 3 工人、1% 概率、数量 1，但当前 `enableSpecialCatch=false`。
- LV2 配置暂与 LV1 相同且关闭，不能作为有效内容。

### 农田

- LV1 岗位上限 2、初始 0 工人。
- 种植、生长、收获、自动收获状态由 `BuildingCropGrowthModule` 模块唯一持有。
- 当前作物配置数组为空，因此系统结构已接入但没有可玩的作物数值；必须先补策划数据再视为内容完成。

### 市场

- 岗位上限 3、初始 3、基础吸引力 100，启用自动补贴并以 3 人为稳定目标。
- 满员时成为优先级 10 的 Resource 提供点；王宫优先级为 0。
- 记录本回合经手资源的 `ItemDefinition.BaseValue × 数量`，回合结束获得总价值的 10% 金币，向下取整。

### 泥路

- 1×1，占用陆地格，移动阻力 5。
- 无运营模块、无升级；道路玩法由网格和寻路系统读取家族静态配置。

### 树木

- 一个家族、一个 Runtime Prefab、八个可建样式：`tree_01`～`tree_08`。
- 放置支付 10 金币、1 树苗。
- 初始生命随机 3～6，双击每次造成 1 点伤害；拆除时获得 2 原木、3 树苗。
- 样式与生命进入实例存档。当前普通拆除也会发放奖励，这是旧行为的原样迁移；是否限制为只有采集至 0 才奖励，列在数值文档待策划项。
- 当前八个 Style 均缺 View 和图标，使用统一占位表现，详见回填指南。

## 4. 施工、升级与表现

- 每个玩家建造实例默认先进入施工阶段；空施工费用也至少有一个施工回合。
- 施工结束不会替换为 LV1 Prefab，而是同实例切换 Stage。
- 升级统一走 `BuildingUpgradeService`，目标等级未配置、科技不满足或费用不足时不修改任何状态。
- 升级保留全部模块与家族状态；等级配置应用后只钳制超出新上限的值。
- Runtime Prefab 不含任何施工/等级美术。表现由 Presentation 的施工 View、可选放置预览 View 和运营 Level/Style 映射动态加载。
- 缺少某级美术时相同样式回退到较低等级；缺少全部美术时显示占位，不影响真实等级和玩法。

## 6. 验证与排错

新增和维护建筑统一使用 Unity 菜单 `Landsong/Building/建筑编辑器`。窗口会一次创建并连接 Family、ModuleSet、Presentation、统一 Runtime Prefab 和 Catalog，不生成脚本；完整流程见《建筑编辑器窗口规划与使用》。

执行 Unity 菜单 `Landsong/Building/Validate Final Architecture`。结构错误必须为零；“尚未回填 View Prefab”是允许的内容警告。

常见问题：

- 建筑菜单没有条目：检查 Family 是否加入 Catalog、蓝图是否锁定、可见条件和 `isDevelopmentCompleted`。
- 无法升级：检查目标等级 `configured`、科技条件和目标级费用。
- 升级后数值没变：检查目标级 LevelConfiguration 是否存在并与 ModuleSet 的 ModuleId 对应。
- 建筑是占位图：不是运行时错误，按回填指南补 Presentation 映射。
- 农田无作物：当前 ModuleSet 的作物数组确实为空，需要内容配置。
