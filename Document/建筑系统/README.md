# Landsong 建筑系统交接总览

> 交接版本：2026-07-13  
> 状态：终态架构、现有建筑迁移、创建编辑器、架构校验器和交接文档均已完成。

## 1. 当前结论

建筑系统已经固定为“一个家族定义 + 一个 ModuleSet + 一个 Presentation + 一个轻量 Runtime Prefab”。所有建筑直接使用密封的 `BuildingBase`；施工和 LV1～LVN 是同一实例的状态，玩法差异由模块承担，表现资源独立加载。

仓库、云中城、高等级正式数值、农田作物、捕鱼后续内容和缺失美术属于内容生产待办，不是架构或交接阻塞项。树木普通拆除是否发放采集奖励仍是明确记录的策划决策项。

## 3. 权威顺序

发生冲突时按以下顺序处理：

1. 当前代码、Family/ModuleSet/Presentation 资产和 Runtime Prefab。
2. 本目录中的《建筑扩展规则》和《AI 添加建筑规则》。
3. 《建筑说明》与《建筑数值策划表》的当前资产快照。
4. 终态决策和历史研究文档。

策划配置中的物品统一引用 `ItemDefinition`；只有库存 API、运行时 DTO、字典键和存档使用由其派生的 ItemId。

## 4. 配置与模块关联

LevelConfiguration 通过自己的 `Apply(BuildingBase)` 明确取得目标模块类型，例如资源产出配置调用 `GetRequiredModule<BM_资源产出>()`。运行时匹配依据是模块 C# 类型，不是 `ConfigurationId` 字符串；`ModuleId` 用于唯一性、顺序、存档和编辑期依赖校验。

新增等级配置时，必须同时保证目标模块存在、启用且顺序正确。当前校验器会检查已登记配置与 ModuleId 的依赖关系。

## 5. 完成修改后的验证

1. 编译运行时与 Editor 程序集。
2. 执行 Unity 菜单 `Landsong/Building/Validate Final Architecture`。
3. 在 Play Mode 验证建造、施工完成、升级、主要模块回合行为和保存/读取。
4. 数值变化同步《建筑数值策划表》，美术变化同步回填指南。
