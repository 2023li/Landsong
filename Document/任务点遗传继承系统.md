实现一个「遗传继承系统」。
系统目标：玩家扮演的不是某一位固定国王，而是一个持续传承的王族血脉。王国会经历多代国王，每一任国王都有自己的寿命、特性、继承人和统治效果。当国王死亡、退位或发生特殊事件时，系统会从继承人中选择新国王，王国进入下一代统治。
一、核心概念
1. 王族血脉
玩家控制的是一个家族，而不是单个角色。
每一代国王只是家族血脉中的一个统治者。
国王死亡后，王位会交接给继承人，游戏继续进行。
2. 国王
每位国王拥有自己的基础数据：
- 唯一 ID
- 姓名
- 年龄
- 最大寿命
- 当前统治回合数
- 父亲
- 母亲
- 王后
- 子嗣列表
- 当前特性列表
- 隐藏特性列表
- 后天获得特性列表
- 当前状态：在位、退位、死亡等
3. 王后
王后会参与子嗣特性的生成。
王子的特性由国王和王后共同决定。
王后也应拥有自己的特性数据，但第一版可以只实现影响遗传的特性，不一定需要完整政治系统。
4. 王子 / 继承人
王子是潜在继承人。
王子出生时会根据父母特性生成自己的潜在特性。
部分特性出生时直接显示，部分特性需要成长到一定年龄或满足条件后才会显示。
王子成年后可以成为合法继承人。
二、特性系统
每个角色可以拥有多个特性。
特性分为先天特性、隐藏特性和后天特性。
1. 先天特性
出生时根据父母特性、血脉规则、随机权重生成。
例如父亲拥有“长寿”，母亲也拥有健康类特性，则王子更容易继承长寿相关特性。
2. 隐藏特性
有些特性不是王子出生时就能显示。
这些特性会先以隐藏状态存在，玩家暂时看不到具体内容，或者只显示为“未知特性”。
达到一定年龄、回合数、事件、教育条件、建筑条件或特殊检测条件后，隐藏特性才会显示或激活。
3. 后天特性
角色在成长、统治、事件、战争、学习、疾病、灾害、政治选择中可能获得新的特性。
后天获得的特性也允许遗传。
例如某位国王在统治期间获得“贤明统治者”，那么他的后代可能继承相关的智慧类潜质。
4. 可遗传特性
不是所有特性都必须遗传。
每个特性需要配置是否可遗传，以及遗传权重。
例如：
- 长寿：可遗传
- 知天命：可遗传或稀有遗传
- 临时疾病：默认不可遗传
- 战争创伤：默认不可遗传
- 贤明统治者：可以作为后天可遗传特性
三、示例特性
1. 长寿
效果：国王最大寿命增加。
例如最大寿命 +20 回合。
该特性可以遗传。
2. 知天命
效果：当国王距离寿命终点还剩 15 个回合时，系统提前提示玩家。
该提示可以用于提醒玩家准备继承、安排继承人、调整王国政策。
该特性可以遗传，但可以设置为较低概率遗传。
3. 早慧
效果：王子较早显示隐藏特性，或成长速度更快。
可以遗传。
4. 体弱
效果：最大寿命减少，或更容易触发疾病事件。
可以遗传。
5. 英武
效果：成为国王后，军事相关收益提高。
例如士兵训练速度 +5%，军队战斗力 +5%。
可以遗传。
6. 博学
效果：成为国王后，科技研究速度提高。
可以遗传。
7. 贪婪
效果：税收提高，但民心下降。
可以是先天特性，也可以后天获得，并允许遗传。
四、遗传规则
王子出生时，系统需要根据父亲和母亲的特性生成王子的潜在特性。
建议每个特性数据包含：
- TraitId：特性 ID
- TraitName：特性名称
- TraitType：特性类型，例如寿命、财政、军事、学术、性格、隐藏、负面等
- IsHeritable：是否可遗传
- InheritWeight：基础遗传权重
- RevealType：显示方式，出生显示 / 年龄显示 / 条件显示 / 事件显示
- RevealCondition：显示条件
- IsAcquired：是否为后天特性
- CanBeInheritedAfterAcquired：后天获得后是否允许遗传
- EffectList：特性效果列表
- ConflictTraits：互斥特性
- RequiredTraits：前置特性
- Rarity：稀有度
遗传时需要考虑：
1. 父亲拥有的可遗传特性。
2. 母亲拥有的可遗传特性。
3. 父母双方是否拥有相似类型特性。
4. 特性的稀有度。
5. 特性之间是否互斥。
6. 是否允许变异，生成父母都没有的新特性。
7. 后天获得特性是否允许进入遗传池。
示例规则：
如果父亲拥有“长寿”，王子有一定概率继承“长寿”。
如果父亲和母亲都拥有寿命类正面特性，则王子继承寿命类特性的概率提高。
如果王子已经获得“长寿”，则不能同时获得互斥的“短命”。
如果父母拥有某些高级特性，王子可能获得隐藏潜质，成年后才显示。
五、隐藏特性显示规则
特性需要支持以下状态：
1. 未生成
角色没有该特性。
2. 已生成但隐藏
角色已经拥有该特性，但玩家暂时不知道。
3. 已发现
玩家知道该特性存在，但效果可以选择是否立刻生效。
4. 已激活
该特性正式参与效果结算。
示例：
王子出生时系统生成了隐藏特性“知天命”，但玩家看不到。
王子成年后，该特性显示为“知天命”。
当他成为国王，并且寿命距离结束还剩 15 回合时，系统触发提示。
六、王位继承规则
当国王死亡或退位时，系统需要选择新国王。
第一版可以采用简单规则：
1. 从合法继承人列表中选择最优先的继承人。
2. 合法继承人必须存活，并达到指定年龄或回合数。
3. 默认优先长子继承。
4. 如果没有成年继承人，可以进入摄政状态，或选择年龄最大的王子临时继承。
5. 如果没有继承人，可以触发王朝危机事件。
继承发生时：
- 当前国王状态变为死亡或退位。
- 新继承人成为国王。
- 新国王的特性开始对王国产生效果。
- 系统记录家族世代表。
- 可以触发继承事件、民心变化、派系反应等扩展逻辑。
七、国王特性效果
国王的特性会影响整个王国。
效果系统应尽量复用王国人才系统中的 Buff / Effect 结构。
示例触发时机：
- OnTurnEnd：每回合结束
- OnKingAscend：新国王登基
- OnKingDeath：国王死亡
- OnPrinceBorn：王子出生
- OnTraitRevealed：特性显现
- OnWarStarted：战争开始
- OnResearchCompleted：研究完成
- OnTaxCollected：税收结算
示例效果：
- 增加最大寿命
- 提高金币收益
- 提高粮食产量
- 提高研究速度
- 提高军队战斗力
- 降低民心
- 增加叛乱风险
- 提前提示寿命终点
八、第一版需要完成的功能
第一版请优先实现以下内容：
1. 王族角色数据结构
支持国王、王后、王子、父母关系、年龄、寿命、状态、特性列表、隐藏特性列表。
2. 特性数据结构
支持是否遗传、遗传权重、是否隐藏、显示条件、是否后天获得、是否可后天遗传、效果列表、互斥特性。
3. 王子出生逻辑
根据国王和王后的特性生成王子的先天特性和隐藏特性。
4. 特性显示逻辑
每回合检查隐藏特性是否满足显示条件。
满足条件后，隐藏特性变为可见或激活。
5. 后天特性获得逻辑
支持通过事件、统治行为、等级、战争、科技、建筑等方式给角色添加后天特性。
后天特性可以配置是否允许遗传。
6. 寿命与死亡逻辑
每回合推进国王年龄或统治回合。
当达到寿命终点时触发死亡或退位。
拥有“长寿”等特性时会影响寿命。
拥有“知天命”时，在死亡前 15 回合提示玩家。
7. 王位继承逻辑
国王死亡后，从继承人中选择新国王。
新国王登基后，他的特性开始影响王国。
8. 王国 Buff 接入
国王特性产生的 Buff 应接入统一效果系统，方便和人才系统、科技系统、建筑系统共同结算。
九、设计要求
系统必须数据驱动、模块化、可扩展。
不要把“长寿”“知天命”等具体特性写死在逻辑中。
应通过 Trait 数据配置不同特性的效果、显示条件、遗传概率和触发时机。
后续系统可能扩展：
- 多个子嗣
- 王后选择
- 公主继承
- 摄政系统
- 继承战争
- 王族分支
- 血脉纯度
- 家族诅咒
- 特性变异
- 教育系统
- 婚姻系统
- 王朝声望
- 继承法修改

## 需要我在 Unity 中完成的部分

1. GameSystem 组件绑定
- Royal Trait Catalog：创建并绑定 RoyalTraitCatalog 资产。
- Royal Inheritance Config：配置初代国王、初代王后、成年年龄、王子基础寿命、每回合出生概率、最大子嗣数量、变异概率。
- 初代国王和王后的 Starting Traits 可直接拖 RoyalTraitDefinition。

2. 王族特性数据资产
- 创建 RoyalTraitDefinition 资产，路径建议放在 `Assets/Landsong/Objects/SO/Inheritance/`。
- 每个 RoyalTraitDefinition 至少配置：Trait ID、Trait Name、Trait Type、Rarity、Heritable、Inherit Weight、Reveal Type、Effects。
- 寿命类特性用 TalentEffectDefinition 的 Effect Type = ModifyMaxLifespan。
- 知天命类预警特性用 Effect Type = LifetimeWarning，Value 配置提前多少回合提示。
- 回合末资源/研究点/蓝图效果可继续用 AddItem、AddResearchPoints、UnlockBuildingBlueprint。
- 创建 RoyalTraitCatalog，把普通可继承特性放入 Traits，把可能突变的特性放入 Mutation Traits。

3. Panel_Game 预制体
- 新增一个继承面板物体，挂 `GamePanel_Inheritance`。
- 在 `UIPanel_Game` 上绑定 `inheritancePanel` 字段。
- HUD 上如需按钮入口，新增或复用一个按钮并绑定 `GamePanel_HUD.btn_继承`。

4. GamePanel_Inheritance 主面板字段
- Close Button：关闭继承面板。
- Birth Prince Button / Birth Prince Button Label：手动生成王子，用于调试或主动生育。
- Abdicate Button / Abdicate Button Label：当前国王退位并触发继承。
- Status Label：显示操作结果。
- Generation Label：显示当前世代和成年年龄。
- Current King Label：显示当前国王摘要。
- Current Queen Label：显示当前王后摘要。
- Heir Root / Heir Item Prefab / Heir Empty Root / Heir Empty Label：继承人列表。
- Character Root / Character Item Prefab / Character Empty Root / Character Empty Label：王族世代表列表。

5. 王族角色条目 prefab
- 挂 `GamePanel_InheritanceCharacterItem`。
- 绑定 Status Marker、Title Label、Meta Label、Age Label、Reign Label、Relation Label、Traits Label。
- 可按需要设置 Reigning Color、Heir Color、Inactive Color。
