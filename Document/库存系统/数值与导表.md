# 库存数值与导表

> 状态：已实现
> 正式工作簿：`ConfigSource/库存系统/库存系统数值表.xlsx`
> 当前 SchemaVersion：`5`
> Unity 菜单：`Landsong/Inventory/库存数值导表工具`

## 1. 权威边界

库存表只管理与具体建筑无关的库存领域数据：

| 库存表拥有 | 建筑表拥有 |
| --- | --- |
| 物品和物品组 | 建筑提供的槽位数量 |
| 槽位类型基础损耗倍率 | 建筑提供的槽位类型 |
| 槽位类型对物品组的损耗修正 | 仓库工人、维护、经验与奖励槽位条件 |
| 自动入库所需的损耗倍率数据 | 住宅消费的物品组与饮食参数 |

库存表不包含 `FamilyId` 或 `Level`，库存导表器不读取和写入建筑家族。

槽位视觉由 UI 代码按 `InventorySlotType` 处理，不从 Excel 导入 Prefab、Sprite、颜色或表现资产。

科技带来的全局损耗修正也不写入库存表。它属于科技系统的持续规则，当前正式入口是建筑工作簿 `科技_全局库存Buff`；库存运行时只接收已经汇总好的全局倍率，不解析 TechnologyId，也不保存科技状态。

## 2. 有效工作表

所有配置表表头在第 4 行，数据从第 5 行开始。

### 2.1 `说明`

`说明!B4` 保存 `SchemaVersion=5`。版本不一致时拒绝导入。

### 2.2 `物品组`

| 字段 | 含义 |
| --- | --- |
| `GroupId` | 稳定物品组 ID |
| `名称` | 显示名 |
| `描述` | 组说明 |
| `ParentGroupId` | 可选父组 |
| `IconAssetPath` | 可选 Sprite 路径；留空保留现有引用 |

当前正式父子关系：

```text
food
└─ food.vegetable
```

导表器校验重复 ID、未知父组、自引用和循环。

### 2.3 `物品`

| 字段 | 含义 |
| --- | --- |
| `ItemId` | 稳定物品 ID |
| `名称`、`描述` | 显示内容 |
| `分类` | `ItemCategory` 枚举名 |
| `可堆叠`、`堆叠上限` | 单槽堆叠规则 |
| `基础价值` | 市场、任务等系统使用 |
| `AddressableKey` | 可选资源键 |
| `标签` | 分号分隔 |
| `每回合损耗率` | 0～1 的物品基础损耗率 |
| `ItemGroupIds` | 分号分隔的物品组 |
| `是否食物` | 是否参与饮食评价 |
| `营养值`、`饮食质量` | 食物属性 |
| `IconAssetPath` | 可选 Sprite 路径；留空保留现有引用 |

### 2.4 `槽位类型`

| 字段 | 含义 |
| --- | --- |
| `SlotType` | `InventorySlotType` 枚举名 |
| `名称` | 策划和 Inspector 显示名 |
| `基础损耗倍率` | 与物品基础损耗率叠乘 |

正式表必须完整覆盖代码中的所有 `InventorySlotType` 枚举值，不能只填写当前已经被建筑使用的类型。

当前工作簿已正式配置：`简陋库存`、`普通库存`、`高级库存`、`冻库`、`粮库`。枚举成员和 Excel 都直接使用中文，不接受旧英文别名。

### 2.5 `槽位分类修正`

| 字段 | 含义 |
| --- | --- |
| `SlotType` | 目标槽位类型 |
| `ItemGroupId` | 目标物品组 |
| `损耗倍率` | 与类型基础倍率继续叠乘 |

当前正式规则：

| SlotType | ItemGroupId | 倍率 |
| --- | --- | --- |
| `冻库` | `food` | `0.80` |
| `粮库` | `food` | `0.60` |

因此冻库格内所有属于 `food` 或其子组的物品，损耗率修正为普通结果的 80%；粮库修正为 60%。

同一物品命中多条组规则时倍率继续叠乘。

### 2.6 自动入库规则

工作簿不再提供“自动存放优先级”字段。运行时把已有同物品堆叠和空槽位放入同一候选集合，始终选择目标物品有效损耗率最低的槽位；同损耗率才优先合并已有堆叠，再按稳定槽位 ID 排序。

工作簿只保留 `说明`、`物品组`、`物品`、`槽位类型`、`槽位分类修正` 五个正式分页。已经迁移或停用的 `槽位表现`、`建筑槽位规则`、`仓储分类修正`、`居民饮食` 分页已删除，不再作为迁移提示保留。

## 4. 导入产物

导表器会创建或更新：

- `ItemDefinition`；
- `ItemGroupDefinition`；
- `InventorySlotTypeDefinition`；
- `ItemCatalog`；
- `InventorySlotTypeCatalog`。

槽位类型资产目录：

```text
Assets/Landsong/Objects/SO/InventorySlotType
```

槽位类型目录：

```text
Assets/Landsong/Objects/SO/InventorySlotTypeCatalog.asset
```

导表器不会：

- 创建或修改建筑家族；
- 创建建筑等级；
- 修改建筑提供的槽位数量或类型；
- 修改住宅饮食参数；
- 创建槽位 UI Prefab 或 Sprite；
- 创建或修改科技全局 Buff；
- 兼容旧库存存档。

## 5. 标准流程

1. 保存并关闭正在编辑的 `库存系统数值表.xlsx`。
2. 打开 `Landsong/Inventory/库存数值导表工具`。
3. 点击“重新读取并校验”。
4. 修复全部错误并检查差异预览。
5. 点击“执行导入”。
6. 如果建筑表引用了新物品组或新槽位类型，再执行建筑数值导入。
7. 执行建筑终态架构校验和 Play Mode 验证。

库存表与建筑表都发生变化时，顺序固定为：

```text
库存表导入 -> 建筑表导入 -> 架构校验
```

若只修改 `科技_全局库存Buff`，无需重新导入库存表，直接执行建筑数值导入即可。

## 6. 校验规则

导表器至少检查：

- 工作簿位于 `ConfigSource/库存系统`；
- Schema 和必需表头一致；
- 稳定 ID 唯一；
- 所有现有物品、物品组都在正式表中；
- 所有 `InventorySlotType` 枚举都有一行正式定义；
- 枚举、布尔值和数值范围合法；
- 物品组父子关系无未知引用和循环；
- 物品引用的组存在；
- 槽位分类修正引用的类型和组存在；
- 相同 `SlotType + ItemGroupId` 不重复；
- 资源路径存在且类型正确。

读取、校验和差异比较期间不会写入任何资产。只有零错误且存在实际差异时才执行增量写入。
