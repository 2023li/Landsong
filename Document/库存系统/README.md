## 维护规则

1. 稳定规则和完整方案写入《库存系统设计与实现》。
2. 每次改变运行时行为、存档结构或默认数值时，同步更新《变更记录》。
3. 尚未实现的内容必须明确标注为“规划”，不能与已落地功能混写。
4. 物品、建筑和 UI 资产的稳定 ID 应写入文档，不依赖文件名表达运行时身份。
5. 不再在 `Document` 根目录或其他目录创建重复的库存系统说明。

## 当前状态

- 当前存档数据版本：`11`。
- 旧库存存档：不兼容、不迁移。
- 当前库存模型：建筑提供槽位，保留单槽堆叠上限。
- 当前槽位身份：`建筑实例 ID + 建筑内本地槽位 ID`。
- 当前职责边界：建筑决定槽位类型与数量；库存决定类型固定规则；UI 决定视觉。
- 当前自然损耗：逐槽计算并分别向下取整。
- 当前科技修正：科技系统提供动态全局倍率；轮子使所有物资库存自然损耗结果乘 `0.90`。
- 当前居民饮食：支持物品分类替代消费和饮食多样性。
- 当前生活质量：首版只纳入饮食因素。
- 当前经济预测：领域服务和基础面板控制器已接入，正式 UI 资产仍需配置。
- 当前正式数值源：`ConfigSource/库存系统/库存系统数值表.xlsx`。
- 当前库存导表 Schema：`3`。
- 当前导表入口：`Landsong/Inventory/库存数值导表工具`。
