# 库存系统变更记录

## 2026-07-17：科技全局库存损耗 Buff

### 最终决策

- 科技影响库存损耗时，通过科技系统的全局 Buff 汇总，不修改每个 `ItemDefinition`、`InventorySlotTypeDefinition` 或建筑等级。
- 库存系统不读取科技树，只在每次损耗计算时使用 `TechnologyGlobalBuffService` 提供的动态倍率。
- 多个全局损耗 Buff 采用乘法叠加。

### 已实现

- 新增 `TechnologyGlobalBuffEffect_InventoryLossReduction`。
- `Inventory` 的最终损耗公式增加科技全局倍率；实际回合结算和经济预测模拟使用同一来源。
- 新增 `buff.inventory.wheel_preservation`，由 `TN_4_5_轮子` 激活，所有物资库存自然损耗降低 10%。
- 建筑工作簿提升为 `SchemaVersion 8`，新增 `科技_全局库存Buff` 工作表。
- 建筑导表器和终态架构校验器已登记库存损耗 Buff 类型及数值范围。

### 验证

- Unity Runtime 与 Editor 程序集编译通过。
- 建筑正式表导入和终态架构校验通过：13 个家族、13 个 Runtime Prefab。
- 100 个基础损耗率 5% 的蔬菜在普通格中：无轮子损耗 5，轮子后有效损耗率 4.5%、实际损耗 4。
- 模拟库存与实际库存的计算结果一致。

## 2026-07-17：槽位类型职责边界定稿

### 最终决策

- 建筑系统管理“提供哪种槽位、多少个槽位”。
- 库存系统管理槽位类型的固定基础倍率、物品组修正和自动存放优先级。
- 建筑缺工、维护失败等状态只产生临时运行时倍率。
- 槽位视觉由 UI 代码按 `InventorySlotType` 映射。
- 库存系统不读取建筑家族或等级，库存表不再出现 `FamilyId + Level`。

### 已实现

- 新增 `InventorySlotTypeDefinition` 和 `InventorySlotTypeCatalog`。
- `InventorySlotProvision` 收口为来源、稳定本地槽位 ID、槽位类型和运行时倍率。
- 移除 `InventorySlotVisualTheme` 和 `InventorySlotPresentationDefinition`。
- 库存工作簿提升为 `SchemaVersion 3`。
- 新增 `槽位类型` 与 `槽位分类修正` 工作表。
- 冻库正式规则设为 `ColdStorage + food + 0.80`。
- 新增父物品组 `food`，`food.vegetable` 改为其子组。
- 库存导表器不再依赖 `BuildingSystem`，不再修改建筑资产。
- 建筑工作簿提升为 `SchemaVersion 7`，为通用库存容量和仓库等级增加槽位类型字段。
- 住宅饮食与生活质量参数迁回建筑数值表。
- 王宫 LV1～LV4 分别使用 `Palace`、`PalaceImproved`、`PalaceAdvanced`、`PalaceRoyal`。
- 仓库 LV1～LV3 分别使用 `BasicWarehouse`、`Warehouse`、`AdvancedWarehouse`。
- 仓库缺工或维护失败时保留物理槽位，只降低保存效果。
- `GameSystem.prefab` 已绑定 `InventorySlotTypeCatalog`。

### 验证

- Unity Runtime 与 Editor 脚本编译通过。
- 库存表和建筑表完成导入，槽位类型资产与 Catalog 已生成。
- 建筑终态架构校验通过：13 个家族、13 个 Runtime Prefab。
- 剩余警告均为允许的美术 View 回填提醒。

## 历史说明

2026-07-16 的 `SchemaVersion 1/2`、按建筑等级配置槽位表现和库存表反向修改建筑等级的方案已经废止。旧工作表只保留迁移提示，不再参与导入。
