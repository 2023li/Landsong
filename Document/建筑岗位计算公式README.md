# 建筑岗位系统计算公式 README

本文档记录当前已采用的岗位系统设计。实现参考：

- `Assets/Landsong/Scripts/BuildingSystem/BuildingJobSystem.cs`
- `Assets/Landsong/Scripts/BuildingSystem/BuildingModules.cs`
- `Assets/Landsong/Scripts/BuildingSystem/Buildings/LumberCabinLV1.cs`

当前用 `LumberCabinLV1` 作为数值样例。

## 岗位吸引力来源

岗位吸引力只来自以下来源：

1. 基础吸引力。
2. 附近人口增益。
3. 全局增益。
4. 全局负增益。

全局增益和全局负增益统一成“全局修正”列表，用正负数区分，并保留来源信息，方便详情栏追踪。

## 总公式

```text
岗位吸引力 = Clamp(
    基础吸引力
    + 附近人口增益
    + 全局修正合计,
    0,
    100
)
```

附近人口增益：

```text
附近人口增益 = 附近人口 * 附近每人口就业吸引力
```

全局修正：

```text
全局修正合计 = Sum(所有全局修正.Value)
```

## 检查器字段单位

附近人口参数不再直接放在 `LumberCabinLV1` 上，而是放在建筑 prefab 的 `BuildingBase` -> `建筑模块` 列表中。
需要附近人口影响岗位吸引力的建筑，添加：

```text
BuildingNearbyPopulationJobAttractionModule
```

```text
人口搜索半径：格，按曼哈顿距离统计。
附近每人口就业吸引力：岗位吸引力点/人。
```

`附近每人口就业吸引力` 越高，每个附近人口提供的岗位吸引力越多。当前没有单独的人口增益上限，最终上限由总岗位吸引力 `Clamp(0, 100)` 控制。

## 全局修正数据

全局修正使用 `BuildingJobAttractionModifier`：

```text
Id: 唯一标识
DisplayName: 详情栏显示名称
Value: 修正值，可正可负
SourceType: 来源类型，例如 Policy、Event、Technology、Difficulty
Description: 可选说明
```

`GameSystem` 当前实现了 `IBuildingJobAttractionModifierProvider`，但暂时返回空列表。之后 Buff 系统接入时，从：

```csharp
GameSystem.GetJobAttractionModifiers(BuildingBase building)
```

返回可溯源修正列表即可。

## 稳定工人

稳定工人公式保留当前分档：

```text
稳定工人 = Clamp(Floor(岗位吸引力 * (最大岗位 + 1) / 100), 0, 最大岗位)
```

以伐木小屋 LV1 为例：

```text
最大岗位 = 3
最低生产工人 = 2
满额生产工人 = 3
```

| 岗位吸引力 | 稳定工人 | 木屋产出 |
| ---: | ---: | --- |
| `0 - 24.99` | `0` | `原木 x0` |
| `25 - 49.99` | `1` | `原木 x0` |
| `50 - 74.99` | `2` | `原木 x1` |
| `75 - 100` | `3` | `原木 x2` |

当前 `1.67` 来自之前方案的等价换算：

```text
25 点人口增益 / 15 人 = 1.67 点/人
```

## 招工与离职

缺工比例：

```text
缺工比例 = 最大岗位 <= 0 ? 0 : (稳定工人 - 当前工人) / 最大岗位
```

自然招工概率：

```text
招工概率 = Clamp(40 + 岗位吸引力 * 0.4 + 缺工比例 * 30, 20, 95)
```

超员比例：

```text
超员比例 = 最大岗位 <= 0 ? 0 : Max(0, 当前工人 - 稳定工人) / 最大岗位
```

自然离职概率：

```text
离职概率 = Clamp(超员比例 * 60 + (100 - 岗位吸引力) * 0.2, 5, 70)
```

每回合规则：

- 当前工人低于稳定工人时，最多自然招 `1` 个工人。
- 当前工人高于稳定工人时，最多自然离职 `1` 个工人。
- 当前工人等于稳定工人时，不变。

## 立即招工费用

```text
立即招工费用 = Ceil(
    实际招工人数 * 单人招工费用 * (1 + (100 - 岗位吸引力) / 100)
)
```

`实际招工人数` 会受最大岗位和可用人口限制。

伐木小屋 LV1 当前：

```text
单人招工费用 = 10
```
