# 建筑岗位计算公式 README

本文档整理当前建筑岗位系统的计算逻辑，来源于：

- `Assets/Landsong/Scripts/BuildingSystem/BuildingJobSystem.cs`
- `Assets/Landsong/Scripts/BuildingSystem/Buildings/LumberCabinLV1.cs`

当前只有 `LumberCabinLV1` 接入了岗位计算、补贴、招工、离职和产出逻辑。后续如果其他建筑也实现 `IBuildingJobSource`，会共用 `BuildingJobSystem` 里的通用公式。

## 当前调整结果

- 已取消附近岗位竞争惩罚。
- 现在不会再扫描附近岗位建筑，也不会因为其他岗位建筑靠近而降低吸引力。
- 保留了外部吸引力惩罚接口，后续全局 Buff 或 Debuff 可以通过接口注入惩罚。
- 人口密度影响从硬编码 `人口密度 * 30` 改成可配置的 `人口密度 * 人口密度吸引力倍率`。
- 木屋当前默认 `人口密度吸引力倍率 = 300`。

## 外部惩罚接口

岗位系统预留了接口：

```csharp
public interface IBuildingJobAttractionPenaltyProvider
{
    float GetJobAttractionPenalty(BuildingBase building);
}
```

当前 `GameSystem` 已实现这个接口，木屋计算时会把 `GameSystem` 作为外部惩罚来源传入：

```csharp
externalAttractionPenalty = BuildingJobSystem.ResolveExternalAttractionPenalty(
    this,
    GameSystem);
```

目前 `GameSystem.GetJobAttractionPenalty(...)` 返回 `0`。后续如果做全局 Buff/Debuff，可以让 `GameSystem` 在这个方法里转发到真正的 Buff 服务。

## 关键结论

- 当前木屋默认基础吸引力是 `35`，最大岗位是 `3`。
- 取消竞争惩罚后，附近其他岗位建筑不会再扣吸引力。
- 半径 `10` 的单格建筑搜索范围理论上约 `221` 个格子。
- 木屋附近 `10` 人时，人口加成约 `13.57`，总吸引力约 `48.57`，仍然稳定 `1` 人。
- 木屋附近约 `12` 人时，总吸引力约 `51.29`，可以稳定 `2` 人，达到基础生产门槛。
- 木屋附近约 `30` 人时，总吸引力约 `75.72`，可以稳定 `3` 人，达到满额生产门槛。

## 通用输入规整

`BuildingJobCalculationInput` 会先把输入修正到合法范围：

```text
最大岗位 = Max(0, maxWorkers)
当前工人 = Clamp(currentWorkers, 0, 最大岗位)
基础吸引力 = Max(0, baseAttraction)
附近人口 = Max(0, nearbyPopulation)
人口格子数 = Max(0, populationCellCount)
人口密度吸引力倍率 = Max(0, populationDensityAttractionMultiplier)
每回合补贴 = Max(0, subsidyGoldPerTurn)
外部吸引力惩罚 = Max(0, externalAttractionPenalty)
单人招工费用 = Max(0, singleRecruitCost)
```

如果 `最大岗位 <= 0`，岗位相关结果基本都会归零。

## 附近人口统计

人口统计分成两个值：

```text
人口格子数 = CountPopulationCells(source, populationSearchRadius)
附近人口 = CountNearbyPopulation(source, buildings, populationSearchRadius)
```

### 人口格子数

`CountPopulationCells` 会从建筑占地的每个格子出发，统计曼哈顿距离小于等于半径的所有格子：

```text
Abs(dx) + Abs(dy) <= radius
```

如果建筑绑定了 `GridMap`，没有基础地块的格子会被跳过。多个占地格子的搜索结果会用 `HashSet` 去重。

单格建筑在没有边界裁剪时：

```text
radius = 10
人口格子数 = 1 + 2 * radius * (radius + 1)
          = 1 + 2 * 10 * 11
          = 221
```

### 附近人口

`CountNearbyPopulation` 会扫描当前建筑列表，只统计满足以下条件的建筑：

- 不是自己。
- 建筑启用。
- 没有拆除中。
- 已放置。
- 与当前建筑在同一个 `GridMap`，或者双方至少一方没有 `GridMap`。
- 实现了 `IBuildingPopulationSource`。
- 两个建筑占地之间的最短曼哈顿距离小于等于搜索半径。

统计结果是这些人口建筑的：

```text
附近人口 += Max(0, populationSource.CurrentPopulation)
```

## 吸引力公式

人口密度：

```text
人口密度 = 人口格子数 <= 0 ? 0 : 附近人口 / 人口格子数
```

人口吸引力加成：

```text
人口吸引力加成 = 人口密度 * 人口密度吸引力倍率
```

人均补贴：

```text
人均补贴 = 最大岗位 <= 0 ? 0 : 每回合补贴 / 最大岗位
```

补贴加成：

```text
补贴加成 = Clamp(人均补贴 * 5, 0, 100)
```

原始岗位吸引力：

```text
原始岗位吸引力 = 基础吸引力 + 人口吸引力加成 + 补贴加成 - 外部吸引力惩罚
```

最终岗位吸引力：

```text
岗位吸引力 = Clamp(原始岗位吸引力, 0, 100)
```

## 稳定工人公式

```text
稳定工人 = 最大岗位 <= 0
    ? 0
    : Clamp(Floor(岗位吸引力 * (最大岗位 + 1) / 100), 0, 最大岗位)
```

以当前木屋 `最大岗位 = 3` 为例：

| 岗位吸引力范围 | 稳定工人 |
| --- | ---: |
| `0 <= 吸引力 < 25` | `0` |
| `25 <= 吸引力 < 50` | `1` |
| `50 <= 吸引力 < 75` | `2` |
| `75 <= 吸引力 <= 100` | `3` |

这意味着木屋想要达到最低生产工人 `2`，岗位吸引力至少要到 `50`。

## 招工和离职概率

缺稳定工人：

```text
缺稳定工人 = Max(0, 稳定工人 - 当前工人)
```

缺满编工人：

```text
缺满编工人 = Max(0, 最大岗位 - 当前工人)
```

缺工比例：

```text
缺工比例 = 最大岗位 <= 0 ? 0 : (稳定工人 - 当前工人) / 最大岗位
```

注意：这里没有再做 `Max(0, ...)`，所以当前工人超过稳定工人时，缺工比例会是负数。不过自然招工只在 `当前工人 < 稳定工人` 时使用。

超员比例：

```text
超员比例 = 最大岗位 <= 0 ? 0 : Max(0, 当前工人 - 稳定工人) / 最大岗位
```

自然招工概率：

```text
招工概率 = 最大岗位 <= 0
    ? 0
    : Clamp(40 + 岗位吸引力 * 0.4 + 缺工比例 * 30, 20, 95)
```

自然离职概率：

```text
离职概率 = 最大岗位 <= 0
    ? 0
    : Clamp(超员比例 * 60 + (100 - 岗位吸引力) * 0.2, 5, 70)
```

每回合处理规则：

- 如果 `当前工人 < 稳定工人`，并且还有可用人口，则按招工概率尝试增加 `1` 个工人。
- 如果 `当前工人 > 稳定工人`，则按离职概率尝试减少 `1` 个工人。
- 如果 `当前工人 == 稳定工人`，本回合工人数不变。

## 可用人口公式

```text
可用人口 = Max(0, 王朝总人口 - 当前所有岗位建筑的当前工人数)
```

当前已雇佣人口通过 `BuildingJobSystem.CountCurrentWorkers(buildings)` 计算。它会统计所有启用、未拆除、实现 `IBuildingJobSource` 的建筑当前工人数。

## 立即招工费用

通用计算结果里的立即招工费用默认按“补满到最大岗位”计算：

```text
立即招工费用 = Ceil(
    缺满编工人 * 单人招工费用 * (1 + (100 - 岗位吸引力) / 100)
)
```

`LumberCabinLV1.TryRecruitImmediately()` 实际执行时会先按可用人口裁剪招工数量：

```text
实际招工人数 = Min(缺满编工人, 可用人口)
实际招工费用 = Ceil(
    实际招工人数 * 单人招工费用 * (1 + (100 - 岗位吸引力) / 100)
)
```

当前木屋默认：

```text
单人招工费用 = 10
```

如果岗位吸引力是 `35`，每个立即招工成本约为：

```text
Ceil(1 * 10 * (1 + 65 / 100)) = 17
```

## 木屋当前默认数值

`LumberCabinLV1` 当前默认岗位参数：

| 参数 | 默认值 |
| --- | ---: |
| 最大岗位 | `3` |
| 基础吸引力 | `35` |
| 每回合补贴 | `0` |
| 单人招工费用 | `10` |
| 人口搜索半径 | `10` |
| 人口密度吸引力倍率 | `300` |
| 外部吸引力惩罚 | `0` |

当前默认产出参数：

| 参数 | 默认值 |
| --- | ---: |
| 最低生产工人 | `2` |
| 满额生产工人 | `3` |
| 基础产出数量 | `1` |
| 满额产出数量 | `2` |
| 产出物品 | `原木` |
| 支付物品 | `金币` |

产出公式：

```text
如果 当前工人 < 最低生产工人:
    当前产出 = 0
否则如果 当前工人 >= 满额生产工人:
    当前产出 = 满额产出数量
否则:
    当前产出 = 基础产出数量
```

木屋默认就是：

```text
当前工人 < 2 -> 原木 x0
当前工人 = 2 -> 原木 x1
当前工人 = 3 -> 原木 x2
```

## 默认木屋案例

### 案例 1：没有附近人口、没有补贴、没有外部惩罚

```text
最大岗位 = 3
基础吸引力 = 35
附近人口 = 0
人口格子数 = 221
人口密度倍率 = 300
每回合补贴 = 0
外部吸引力惩罚 = 0

人口密度 = 0 / 221 = 0
人口吸引力加成 = 0 * 300 = 0
补贴加成 = 0
原始岗位吸引力 = 35 + 0 + 0 - 0 = 35
岗位吸引力 = 35
稳定工人 = Floor(35 * 4 / 100) = 1
```

结果：

```text
稳定工人 = 1
最低生产工人 = 2
当前稳定状态无法生产
```

### 案例 2：附近有 10 人，半径仍为 10

```text
附近人口 = 10
人口格子数 = 221
人口密度 = 10 / 221 = 0.0452
人口吸引力加成 = 0.0452 * 300 = 13.57
原始岗位吸引力 = 35 + 13.57 = 48.57
稳定工人 = Floor(48.57 * 4 / 100) = 1
```

结果仍然是：

```text
稳定工人 = 1
还差一点达到最低生产工人 2
```

### 案例 3：附近有 12 人，半径仍为 10

```text
附近人口 = 12
人口格子数 = 221
人口密度 = 12 / 221 = 0.0543
人口吸引力加成 = 0.0543 * 300 = 16.29
原始岗位吸引力 = 35 + 16.29 = 51.29
稳定工人 = Floor(51.29 * 4 / 100) = 2
```

结果：

```text
稳定工人 = 2
达到最低生产工人 2
当前预计产出 原木 x1
```

### 案例 4：附近有 30 人，半径仍为 10

```text
附近人口 = 30
人口格子数 = 221
人口密度 = 30 / 221 = 0.1357
人口吸引力加成 = 0.1357 * 300 = 40.72
原始岗位吸引力 = 35 + 40.72 = 75.72
稳定工人 = Floor(75.72 * 4 / 100) = 3
```

结果：

```text
稳定工人 = 3
达到满额生产工人 3
当前预计产出 原木 x2
```

## 当前仍需关注的问题

1. `基础吸引力 35` 与 `最低生产工人 2` 仍然不是天然匹配。
   - 没有附近人口时，木屋稳定工人仍然只有 `1`。
   - 这代表木屋现在依赖人口聚集或补贴来启动生产。

2. 人口密度仍然受搜索半径影响。
   - 半径越大，分母越大。
   - 如果希望“只要附近有住房就明显加成”，应继续降低 `populationSearchRadius`，或改成人口数量加成。

3. 自然招工每回合最多只加 `1` 人。
   - 即使稳定工人已经达到 `2` 或 `3`，也需要多个回合才能满员。

4. 补贴按“总补贴 / 最大岗位”转成人均补贴。
   - 最大岗位越多，同样总补贴越不明显。
   - 补贴要达到明显效果，需要与经济产出一起重新核算。

## 后续调参建议

建议先决定设计目标，再改数值：

| 目标 | 优先调整项 |
| --- | --- |
| 木屋建好后能自然进入基础生产 | 提高 `baseJobAttraction` 到至少 `50`，或把 `minimumWorkersForProduction` 降到 `1` |
| 附近人口对招工更明显 | 提高 `populationDensityAttractionMultiplier`，或降低 `populationSearchRadius` |
| 人口只要靠近就有稳定收益 | 把人口密度公式改成附近人口数量加成 |
| 后续全局 Buff 产生惩罚 | 在 `GameSystem.GetJobAttractionPenalty(...)` 里转发到 Buff 服务 |
| 招工节奏更快 | 允许每回合招多个工人，或提高立即招工的性价比 |

如果只做最小改动，让默认木屋无人口也能工作，最直接的两种方案是：

```text
方案 A：
baseJobAttraction: 35 -> 50
minimumWorkersForProduction: 保持 2

方案 B：
baseJobAttraction: 保持 35
minimumWorkersForProduction: 2 -> 1
```

当前这次调整选择的是第三种方向：

```text
方案 C：
取消附近岗位竞争惩罚
populationDensityAttractionMultiplier: 30 -> 300
```

方案 C 不会让木屋无条件启动生产，但会让附近人口真正影响岗位吸引力。
