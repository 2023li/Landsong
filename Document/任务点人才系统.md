人才系统
实现一个「王国人才系统」。
系统目标：玩家可以通过招募获得不同职业的人才，并将人才放入王国的人才槽中。被任命的人才会为王国提供长期 Buff，例如资源收益、军事能力、研究速度、特殊事件加成等。人才可以升级，并且部分特性一开始不可见，需要满足条件后才会发现或激活。
一、人才职业
第一版暂时只需要支持以下 4 类职业：
1. 大总管
定位类似财政大臣，主要影响金币、税收、利息、维护费、市场收益等。
示例效果：每回合获得当前库存金币 2% 的利息。
2. 大法师
主要影响魔法、元素学、灾害抵抗、特殊资源、幻想建筑等。
示例效果：元素学研究速度 +10%，或降低灾害损失。
3. 大将军
主要影响军队训练、士兵战斗力、城市防御、军队维护费等。
示例效果：士兵训练速度 +10%，城市防御 +10%。
4. 大学者
主要影响科技研究、学院产出、文化、知识点、特殊科技解锁等。
示例效果：科技研究速度 +10%，或每回合额外获得研究点。
二、人才槽规则
王国拥有两类人才槽：
1. 通用人才槽
任何职业的人才都可以放入。
例如：王国默认拥有 3 个通用人才槽。
2. 专属人才槽
只能放入指定职业的人才。
例如：军事专属人才槽只能放入大将军类人才。
后续可以扩展为财政专属槽、法师专属槽、学者专属槽等。
人才只有被放入人才槽后，才会对王国产生主动 Buff。
人才从人才槽移除后，对应 Buff 失效，但人才本身仍保留在人才池中。
三、人才池与费用
玩家招募到的人才会进入人才池。
无论人才处于人才池，还是已经被放入人才槽，只要人才仍属于玩家，都需要持续支付费用。
费用可以按回合结算，例如每回合消耗金币作为薪资或维护费。
如果金币不足，第一版可以先只记录警告或 Debug 日志，后续再扩展人才离开、忠诚度下降、Buff 失效等机制。
四、招募与刷新规则
招募人才的流程如下：
1. 玩家支付金币进行一次人才刷新。
2. 系统生成若干张人才卡片。
3. 玩家从刷新出来的人才卡片中选择想要招募的人才。
4. 被选择的人才进入人才池。
5. 未选择的人才卡片在本轮刷新结束后消失。
刷新人才时需要支持条件筛选。
有些特殊人才不会默认进入普通刷新池，必须满足特定条件后才有概率出现。
例如：
- 建造法师塔后，大法师类高级人才才会刷新。
- 解锁军事科技后，高级大将军才会刷新。
- 王国金币达到一定数量后，高级大总管才会刷新。
- 建造学院后，大学者类人才才会刷新。
- 完成特殊事件后，某个唯一人才才会加入刷新池。
人才数据中需要包含刷新条件。
刷新条件应当数据化配置，不要写死在刷新逻辑中。
五、人才卡片 UI 表现
不同职业的人才在 UI 上需要有明显区分。
例如人才卡片颜色、边框、图标、背景纹理或职业标签不同。
第一版至少需要在数据层支持：
- 人才职业
- 卡片主色
- 职业图标
- 稀有度颜色
- 是否显示隐藏特性
- 当前薪资
- 当前等级
- 当前已解锁 Buff 描述
示例：
- 大总管：偏金色或棕色
- 大法师：偏紫色或蓝色
- 大将军：偏红色或铁灰色
- 大学者：偏青色或绿色
UI 第一版可以先做简单调试界面，也可以通过 Debug 日志验证，但数据结构需要支持正式 UI 后续接入。
六、人才效果系统
每个人才可以拥有多个效果。
效果不要写死在具体人才代码中，应使用统一的 Buff / Effect 系统处理。
建议效果数据结构包含：
- EffectType：效果类型
例如 AddResource、ModifyResourceGainPercent、ModifyCostPercent、ModifyBuildSpeed、ModifyResearchSpeed、ModifyMilitaryPower、ModifyHappiness、ModifyRisk 等。
- TriggerTiming：触发时机
例如 OnTurnEnd、OnResourceProduced、OnBuildingConstructed、OnResearchCompleted、OnTradeCompleted、OnRecruitSoldier 等。
- ValueType：数值类型
例如固定值、百分比、基于库存百分比、基于人口、基于建筑数量、基于等级等。
- Value：基础数值
- LevelScaling：随等级成长的数值
示例：
大总管的效果可以配置为：
TriggerTiming = OnTurnEnd
EffectType = AddResource
ResourceType = Gold
ValueType = PercentOfCurrentStorage
Value = 2%
表示每回合结束时，获得当前库存金币 2% 的额外金币。
七、升级与隐藏特性
人才可以升级。
升级后可以提升已有 Buff 数值，也可以解锁新的效果。
人才还可以拥有隐藏特性。
隐藏特性一开始不完全展示，可以显示为“未知特性”。
隐藏特性需要支持以下状态：
1. 未发现
玩家只能看到该人才可能拥有隐藏特性，但不知道具体内容。
2. 已发现
玩家知道隐藏特性的描述，但效果尚未生效。
3. 已激活
隐藏特性正式生效，并参与 Buff 结算。
隐藏特性可以通过等级、事件、建筑、科技、使用次数、任命时间等条件解锁。
例如：
- 大总管达到 3 级后，发现“贪婪税吏”特性。
- 大法师在法师塔任命 5 回合后，激活“元素共鸣”。
- 大将军经历战争事件后，解锁“百战老兵”。
- 大学者完成一次高级科技研究后，解锁“灵感爆发”。
八、第一版需要完成的功能
第一版请优先实现以下核心流程：
1. 人才数据结构
支持 ID、名称、职业、稀有度、等级、经验、费用、刷新条件、卡片 UI 数据、普通效果、隐藏特性等。
2. 人才刷新系统
玩家消耗金币刷新若干人才卡片。
系统根据当前王国状态、建筑、科技、事件等条件筛选可刷新人才。
玩家可以选择其中一张或多张人才卡片进行招募。
3. 人才池系统
已招募人才进入人才池。
人才池中的人才即使没有被任命，也需要支付费用。
4. 人才槽系统
支持通用人才槽和专属人才槽。
通用槽可以放入任意职业。
专属槽只能放入指定职业。
人才放入槽位后 Buff 生效，移除后 Buff 失效。
5. 费用结算
每回合结算所有已拥有人才的费用。
包括人才池中的人才和已任命到槽位中的人才。
6. Buff 结算
根据人才槽中已任命的人才，在对应触发时机执行效果。
第一版至少需要支持 OnTurnEnd 回合结束效果。
7. 升级系统
人才可以升级，升级后提升效果或解锁新效果。
8. 隐藏特性系统
支持隐藏特性的发现、显示和激活状态。
9. UI 数据支持
不同职业的人才卡片可以显示不同颜色、图标和职业标签。
正式 UI 可以后续完善，但数据层必须支持。
九、设计要求
代码需要模块化、数据驱动、可扩展。
不要为每个具体人才单独写死逻辑。
人才、效果、隐藏特性、刷新条件、槽位限制都应尽量通过数据配置完成。
系统后续可能扩展：
- 更多职业
- 更多人才槽
- 更多专属槽
- 唯一人才
- 忠诚度
- 派系斗争
- 人才死亡或离职
- 人才事件
- 人才组合羁绊
- 人才专属任务
第一版重点是先完成完整主流程：
刷新人才 → 支付金币 → 选择人才卡 → 加入人才池 → 支付持续费用 → 放入人才槽 → Buff 生效 → 升级 → 解锁隐藏特性。
