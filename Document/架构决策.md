# Landsong 正式架构决策

> 冻结日期：2026-07-17。本文记录已经确认的项目级边界；它们是实现规则，不再作为待讨论建议。

## 1. 任务内容使用 Definition/Catalog

决策：静态任务使用独立 `QuestDefinition` 资产，并由唯一 `QuestCatalog` 登记。

- `GameSystem.prefab` 只保存 QuestCatalog 引用、起始任务 ID 和少量全局生成参数。
- `QuestService` 是运行时状态、完成判定、随机生成、事件和存档的唯一所有者。
- 静态任务存档只保存稳定任务 ID 与进度；运行时生成任务额外保存生成定义快照。
- 新增任务不得修改 `GameSystem.prefab` 内嵌数组，因为该数组已经退出架构。

正式资产：`Assets/Landsong/Objects/SO/Quests/QuestCatalog.asset`。

## 2. GameSystem 保持单一 Facade，按领域 partial 拆分

决策：保留一个 `GameSystem` 类型和一个 `GameServices` 公开门面，不引入第二个总控；物理文件按编排职责拆分。

| 文件 | 责任 |
| --- | --- |
| `GameSystem.cs` | 序列化装配、生命周期、核心 Service 创建、建筑/库存/王朝协调 |
| `GameServices.cs` | 唯一公开领域入口 |
| `GameSystem.Turn.cs` | 回合字段、推进、跨领域结算顺序、游戏结束 |
| `GameSystem.Technology.cs` | 科技创建、研究点、完成效果、解锁内容 |
| `GameSystem.Expedition.cs` | 远征创建、结算、消息和全局岗位惩罚 |
| `GameSystem.Talent.cs` | 人才创建、结算和消息 |
| `GameSystem.Inheritance.cs` | 继承创建、结算和消息 |
| `GameSystem.Blueprints.cs` | 建筑蓝图初始化与自动条件协调 |
| `QuestSystem/GameSystem.Quest.cs` | 任务目录引用和 QuestService 装配 |

partial 只解决物理可读性。领域规则仍必须进入 Service、Module 或正式配置，不能借拆文件继续扩大 `GameSystem` 的规则所有权。

## 3. 首个公开版本前严格不兼容旧游戏存档

决策：开发期采用项目级严格版本策略。

- `GameSaveRepository` 只加载 `DataVersion == GameData.CurrentDataVersion` 的存档。
- 当前 `GameData.CurrentDataVersion = 12`。
- 数据结构变化时提升项目级版本；旧存档直接不显示或拒绝加载。
- 子系统不得维护自己的存档版本、迁移链、旧枚举别名或不可达兼容分支。
- `Validate()` 只校验当前结构，不承担旧版本迁移。

切换到公开版本的存档迁移策略必须作为新的项目级决策：届时建立统一 `IGameSaveMigration` 链，由仓储层先迁移整体 `GameData`，禁止各 UI/Service 临时兼容。

## 4. 命名空间采用“新代码强制，旧序列化类型冻结”

决策：

- 新建 C# 类型必须进入职责对应的 `Landsong.*` 命名空间。
- 已在 Scene、Prefab 或 ScriptableObject 中序列化的旧全局 `MonoBehaviour`/`ScriptableObject` 不做批量改名。
- 纯 C#、未被 Unity 序列化引用的旧类型可以在相关功能修改中逐步迁移。
- 若必须迁移序列化类型，需要单独任务提供 Unity 支持的类型迁移属性、全资产重导入和 Scene/Prefab/SO 验证。

目录统一不是破坏 Unity 类型引用的充分理由。

## 5. 配置媒介按领域选择

决策：不追求“所有配置都进 Excel”或“所有配置都进 ScriptableObject”的形式统一。

- 建筑、库存的大规模策划数值使用正式 Excel 导表。
- 地图、科技、政策、任务等结构型内容使用 Definition/Catalog 和 Unity 编辑器。
- UI、表现、动画、音频只存 Unity 资产。
- 每个字段只能有一个权威源，生成资产不得反向覆盖正式源表。

## 6. 逻辑事件与玩家消息保持双轨

决策：

- Service 的强类型事件用于逻辑订阅和 UI 刷新。
- `GameEventService` 只保存玩家可见消息和可点击事件记录。
- 不能通过解析玩家消息驱动玩法，也不能用消息列表替代领域状态。

## 7. 暂不引入 asmdef

决策：当前不新增自研 asmdef。先稳定命名空间、序列化类型和依赖方向；只有当编译隔离收益明确且能给出依赖图、Editor/Runtime 切分及全项目验证方案时再评估。
