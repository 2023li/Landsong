# 玩家自定义语言包制作说明

Landsong 的语言包 V1 只包含文本。语言包不能加载脚本、DLL、Prefab、图片、字体、Addressables 或远程 URL。

## 安装目录

每个语言包使用一个独立目录，放入游戏存档根目录的 `ExternalLanguagePacks`：

```text
ExternalLanguagePacks/
  community.ja/
    manifest.json
    strings.csv
```

目录名不是语言包身份；`manifest.json` 中的 `packId` 才是稳定身份。两个目录声明相同 `packId` 时，两者都会被禁用。

## manifest.json

```json
{
  "schemaVersion": 1,
  "packId": "community.ja",
  "displayName": "日本語",
  "localeCode": "ja-JP",
  "fallbackLocaleCode": "en",
  "author": "Example Author",
  "description": "Japanese community translation",
  "targetKeysetVersion": 1
}
```

- `packId`：2–64 位 ASCII 标识，只能包含字母、数字、点、下划线和连字符。
- `localeCode`：运行平台可识别的 BCP 47 语言标签。
- `fallbackLocaleCode`：V1 只允许 `en` 或 `zh-Hans`。
- `targetKeysetVersion`：当前为 `1`；不一致会提示语言包可能过期。

## strings.csv

文件必须是严格 UTF-8，表头必须包含 `Table,Key,Text`。支持标准 CSV 引号、逗号、双引号转义和字段内换行。

```csv
Table,Key,Text
UI,ui.main_menu.new_game,ニューゲーム
Content,content.item.u044f5e79.name,小麦
Gameplay,gameplay.expedition.ui.success,成功率 {0:0.#}%
```

允许的 Table：

- `UI`：菜单、按钮、面板标题和固定提示；
- `Content`：物品、建筑、科技、任务等正式内容；
- `Gameplay`：状态、事件和动态格式文本。

语言包可以只翻译一部分 Key，缺项自动使用 manifest 指定的内置语言。空文本表示刻意显示为空，不等同于缺项。

## Smart String 规则

动态条目的参数集合必须与内置条目一致。参数可以重排，但不能删除、增加或改名。例如：

```text
内置：预计成功率 {0:0.#}%
合法：成功率は {0:0.#}% です
非法：成功率は {chance}% です
```

非法 Smart String、重复 Key、非法 UTF-8、越界文件或非法 manifest 会产生诊断；结构错误的语言包不会替换当前已成功加载的语言。

## 获取完整 Key

Unity 编辑器菜单 `Landsong/Localization/Export Language Pack Template` 会从 `UI`、`Content`、`Gameplay` 三张正式 StringTable 导出当前完整模板。不要手写或猜测 Key，也不要用中文显示文本、资产名或文件名作为 Key。

## 字体限制

V1 语言包不能携带字体。目标文字能否显示取决于游戏内置 TMP 字体及 fallback 字体覆盖；缺字时请将诊断和目标文字系统反馈给开发者。
