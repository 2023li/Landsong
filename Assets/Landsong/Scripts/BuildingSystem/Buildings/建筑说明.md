## 岗位系统介绍
建筑会根据附近人口、补贴金额和周围岗位竞争计算岗位吸引力。
岗位吸引力越高，建筑越容易招到工人，也越能留住工人。

给建筑设置每回合金币补贴，可以提高岗位吸引力。
补贴越高，建筑越容易稳定满员。

玩家也可以直接花金币立即招工。
但立即招工只是把人招来，并不会提高岗位本身的吸引力。
如果建筑吸引力不足以支撑当前工人数，超出的工人之后仍然可能离职

公式：
人均补贴 = 每回合补贴金币 / 最大岗位数

补贴加成 = Clamp(人均补贴 × 5, 0, 100)

原始岗位吸引力：= 基础吸引力 + 人口密度 × 30 + 补贴加成 - 惩罚项

岗位吸引力 = Clamp(原始岗位吸引力,0, 100)

人口密度：建筑周围半径10格所有的人口/格子数

稳定支撑工人数 = Clamp( Floor(岗位吸引力 × (最大岗位数 + 1) / 100), 0, 最大岗位数 )

缺工比例 = (稳定支撑工人数 - 当前工人数) / 最大岗位数

每回合招工概率 = Clamp( 40 + 岗位吸引力 × 0.4 + 缺工比例 × 30, 20, 95 )

超员比例 = (当前工人数 - 稳定支撑工人数) / 最大岗位数

每回合离职概率 = Clamp( 超员比例 × 60 + (100 - 岗位吸引力) × 0.2, 5, 70 )

立即招工费用 = 缺少工人数 × 单人招工费用 × (1 + (100 - 岗位吸引力) / 100)
### 居民房LV0
脚本名称：ResidentialHousingLV1

建筑效果：
* 提供人口： 最多提供5人口，初始提供2人口
* 消耗资源人口增长： 回合结束时10格范围内若有 "资源提供点" 则每人口消耗1单位“蔬菜”，每3回合 若都满足消耗 则人口+1
* 提供税收：若人口达到上限，每5回合都消耗成功 则提供 人口数的金币
* 连续3个回合消耗失败则人口-1，人口为0时，建筑进入荒废状态
* 消耗失败需要记录 后期会用UI显示

备注：
* "资源提供点"检索范围作为一个可配置变量 
* 税收的时间 作为可配置变量
* 需要保留接口 后期所有税收需要在UI上显示、所有的资源消耗需要在UI上显示
* 寻路采用曼哈顿距离，有道路的格子寻路代价为0.5，若目标建筑不是1*1建筑 只要有一格达到目标 则视为能够接触到目标

### 伐木小屋LV1
脚本名称：LumberCabinLV1

建筑效果：
* 提供岗位：最多提供3个岗位
* 产出木材：每回合如果至少有2个工人，则产出1单位“原木”，若有3个工人则产出2单位原木
* 在建筑概览面板中显示 补贴 当前工人/稳定工人 原始岗位吸引力

备注：
* 需要提供一种调试途径能够查看岗位系统所有的影响因素的值
* 岗位系统的计算可以作为一个静态类复用
